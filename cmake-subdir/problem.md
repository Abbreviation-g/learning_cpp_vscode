# CMake MinGW VS Code Debug Problems

## 1. `cmake --build --preset mingw` failed with MSYS make

### Symptom

The build failed during CMake compiler detection or try-compile:

```text
'mTC_xxxx.dir' is not recognized as an internal or external command
make: *** [cmTC_xxxx/fast] Error 1
```

### Cause

The project used the `MinGW Makefiles` generator, but `CMAKE_MAKE_PROGRAM`
pointed to:

```text
C:/MinGW/msys/1.0/bin/make.exe
```

`MinGW Makefiles` expects MinGW's native make program. MSYS `make.exe` can
mis-handle Windows-style paths and backslashes generated by this generator.

### Fix

Use MinGW's `mingw32-make.exe` with the `MinGW Makefiles` generator:

```json
"generator": "MinGW Makefiles",
"cacheVariables": {
    "CMAKE_CXX_COMPILER": "g++",
    "CMAKE_C_COMPILER": "gcc",
    "CMAKE_MAKE_PROGRAM": "C:/MinGW/bin/mingw32-make.exe"
}
```

After changing the make program, remove the old CMake cache or rebuild the
build directory, because CMake caches `CMAKE_MAKE_PROGRAM`.

```powershell
Remove-Item .\build\mingw -Recurse -Force
cmake --preset mingw
cmake --build --preset mingw
```

## 2. VS Code debugger did not enter breakpoints

### Symptom

VS Code started debugging, but breakpoints were not hit reliably.

### Cause

The CMake preset was using:

```json
"CMAKE_BUILD_TYPE": "Release"
```

Release builds enable optimization and usually do not provide the expected
debug information for source-level debugging.

### Fix

Use a Debug build for VS Code debugging:

```json
"CMAKE_BUILD_TYPE": "Debug"
```

Then regenerate and rebuild:

```powershell
cmake --preset mingw
cmake --build --preset mingw
```

This produces binaries with `-g`, allowing GDB to bind breakpoints to source
locations.

## 3. stdout did not appear in VS Code Terminal while debugging

### Symptom

With this launch configuration:

```json
"externalConsole": false
```

breakpoints worked, but `std::cout` output did not appear in the VS Code
Terminal panel.

### Cause

For `cppdbg` with MinGW GDB on Windows, stdout routing is controlled by the
C++ extension. Even when `externalConsole` is `false`, output may go to the
Debug Console instead of the Terminal panel. This behavior depends on the
VS Code C++ extension, GDB, and Windows console redirection behavior.

### Fix

Keep internal debugging enabled and make program output visible in the Debug
Console:

```json
"externalConsole": false,
"avoidWindowsConsoleRedirection": false,
"internalConsoleOptions": "openOnSessionStart",
"logging": {
    "programOutput": true
}
```

Use the Debug Console panel to view stdout during debugging.

If stdout must appear in a real console window, use:

```json
"externalConsole": true
```

That opens an external console window instead of using VS Code's internal
Terminal panel.

## Final working setup

### `CMakePresets.json`

```json
{
  "version": 3,
  "configurePresets": [
    {
      "name": "mingw",
      "displayName": "MinGW Makefiles",
      "generator": "MinGW Makefiles",
      "binaryDir": "${sourceDir}/build/mingw",
      "cacheVariables": {
        "CMAKE_CXX_COMPILER": "g++",
        "CMAKE_MAKE_PROGRAM": "C:/MinGW/bin/mingw32-make.exe",
        "CMAKE_C_COMPILER": "gcc",
        "CMAKE_BUILD_TYPE": "Debug"
      }
    }
  ],
  "buildPresets": [
    {
      "name": "mingw",
      "configurePreset": "mingw"
    }
  ]
}
```

### `.vscode/launch.json`

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Debug cmake_subdir_demo",
            "type": "cppdbg",
            "request": "launch",
            "program": "${workspaceFolder}\\build\\mingw\\src\\cmake_subdir_demo.exe",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": false,
            "avoidWindowsConsoleRedirection": false,
            "internalConsoleOptions": "openOnSessionStart",
            "logging": {
                "programOutput": true
            },
            "MIMode": "gdb",
            "miDebuggerPath": "C:\\MinGW\\bin\\gdb.exe",
            "preLaunchTask": "CMake: build mingw",
            "setupCommands": [
                {
                    "description": "Enable pretty-printing for gdb",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ]
        }
    ]
}
```
